01 — Algorithme
Comprendre Dinic
L'algorithme de Dinic trouve le flot maximum par recherche itérative de chemins augmentants dans un graphe résiduel, en utilisant un BFS pour garantir les plus courts chemins.

Réseau R1 : 5 sommets, 7 arcs. Source = 1, Puits = 5.

Itération 1

Itération 2

BFS trouve le chemin 1 → 2 → 3 → 5. Capacité minimale = 10.

chemin : 1→2→3→5 flot cumulé = 30

Itération 3

BFS trouve le chemin 1 → 2 → 4 → 5. Capacité minimale = 10.

chemin : 1→2→4→5 flot cumulé = 40

Terminaison

Plus aucun chemin augmentant trouvé par BFS. L'algorithme s'arrête.

flot maximum = 40
💡 Pourquoi le BFS ?

L'exploration en largeur garantit que le premier chemin trouvé est minimal en nombre d'arcs. Chaque sommet est visité une seule fois, donnant une complexité O(n+m) par phase. Cela assure la convergence rapide de l'algorithme en O(V²·E).

02 — Structures de données
Analyse comparative
Trois structures ont été évaluées pour représenter le réseau. La liste de successeurs s'impose comme le meilleur choix.
📊

Matrice d'incidence

Mémoire O(n×m). Accès successeurs O(m). Ajout/suppression O(n). Trop coûteuse pour les graphes creux.

📋

Tableau de sommets

Mémoire O(n+m). Accès O(deg⁺). Ajout O(1), suppression O(deg⁺). Bonne alternative, mais moins flexible.

🔗

Listes de successeurs ✓

Mémoire O(n+m). Accès O(deg⁺). Ajout O(1), suppression O(deg). Gagnante sur tous les critères.

CritèreMatrice d'incidenceTableau de sommetsListe de successeurs
MémoireO(n × m)O(n + m)O(n + m) ✓

Ajout d'arcO(n)O(1)O(1) ✓

🔧 Adaptation du graphe résiduel

Pour chaque arc (u,v) du réseau initial, on crée deux arcs dès le départ : un arc direct de capacité c(u,v)−f(u,v) et un arc inverse de capacité f(u,v). Ainsi, on ne fait jamais d'ajout/suppression dynamique — seulement des mises à jour de capacités.

Implémentation C
Le code complet : structures de données (dinic.h) et implémentation (dinic.c).
dinic.h
#ifndef DINIC_H
#define DINIC_H
#include <stdbool.h>

// Maillon : arc sortant dans la liste chaînée
typedef struct maillon {
    int SommetDest;        // sommet destination
    int capa;              // capacité résiduelle
    int flot;              // flot courant
    struct maillon *suivant; // arc suivant
} Maillon;

// Liste chaînée des successeurs d'un sommet
typedef struct listeSucc {
    Maillon *tete;
    int      taille;
} ListeSucc;

// Chemin source → puits
typedef struct chemin {
    int *sommets;
    int  taille;
} Chemin;

// Graphe orienté (réseau ou résiduel)
typedef struct graphe {
    int       nbSommets;
    ListeSucc *successeurs;
    int       source;
    int       puits;
} Graphe;

// Prototypes
Graphe* buildGraph(const char *filename);
Graphe* buildResidualGraph(const Graphe *g);
Chemin* shortestPath(const Graphe *rg);
int     minCapacity(const Graphe *rg, const Chemin *c);
void    updateResidualGraph(Graphe *rg, const Chemin *c, int k);
void    updateNetworkFlow(const Graphe *rg, Graphe *g);
void    ecrireResultat(const Graphe *res, int flot);
void    freeGraph(Graphe *g);
void    freePath(Chemin *c);

#endif
dinic.c — main + pagerank
// Algorithme principal de Dinic
int main(int argc, char **argv) {
    if (argc != 2) {
        fprintf(stderr, "Usage: %s <DIMACS>\n", argv[0]);
        return EXIT_FAILURE;
    }
    Graphe *network = buildGraph(argv[1]);
    Graphe *resGraph = buildResidualGraph(network);

    int maxFlow = 0;
    Chemin *path;

    // Boucle principale : tant qu'un chemin augmentant existe
    while ((path = shortestPath(resGraph))) {
        int k = minCapacity(resGraph, path);
        updateResidualGraph(resGraph, path, k);
        updateNetworkFlow(resGraph, network);
        maxFlow += k;
        freePath(path);
    }

    ecrireResultat(network, maxFlow);
    freeGraph(resGraph);
    freeGraph(network);
    return EXIT_SUCCESS;
}
